## TOP 3 상세 분석

### 1위: 내주변 맛집&카페 추천

> 예시: "나는 지금 ㅇㅇ지역에 있는데 이 주변 맛집 30대초반 한식 추천, 식사 후 주변 분위기 좋은 카페 or 소품샵까지 데이트코스 추천"

#### 장점
- Naver Place API, Kakao Map API **공식 지원**
- 리뷰 데이터 풍부하고 크롤링 용이
- 위치 기반 검색 구현 쉬움
- 데이트코스 연계로 확장성 좋음

#### 데이터 소스
| 소스 | 수집 방법 | 데이터 |
|------|----------|--------|
| Naver Place API | 공식 API | 기본 정보, 위치, 영업시간, 카테고리 |
| 네이버 플레이스 | 크롤링 | 리뷰 텍스트, 별점, 방문자 리뷰 |
| Kakao Map API | 공식 API | 위치 검색, 길찾기, 주변 검색 |

#### MVP 범위
- 지역: 1-2개 (예: 강남, 홍대)
- 카테고리: 한식, 양식, 카페, 소품샵
- 기능: 위치 입력 → 맛집 추천 → 근처 카페 연계

---

### 2위: 컨디션 기반 메뉴 추천

> 예시: 기분/피로/숙취/생리/감기 등 컨디션 기반으로 "지금 내 컨디션에 맞는 메뉴" 추천

#### 장점
- 기존 "오늘 뭐 먹지"와 **명확한 차별점**
- 리뷰에서 "맵다/자극적/든든/깔끔" 키워드 추출 → NLP로 구현 용이
- 채팅 인터페이스로 UX 직관적

#### 데이터 소스
| 소스 | 수집 방법 | 데이터 |
|------|----------|--------|
| 네이버 플레이스 | 크롤링 | 메뉴별 리뷰, 맛 표현 키워드 |
| 배달의민족/요기요 | 크롤링 | 배달 메뉴 리뷰, 별점 |
| 자체 태깅 | 수동/반자동 | 컨디션-메뉴 매핑 데이터 |

#### MVP 범위
- 지역: 1-2개
- 메뉴 카테고리: 8-10개
- 컨디션: 피로, 숙취, 스트레스, 감기, 다이어트 등 5-6개

#### 핵심 AI 기능
1. 채팅으로 컨디션 파악
2. 리뷰에서 맛/식감 표현 추출 (NLP)
3. 컨디션 ↔ 메뉴 특성 매칭

---

### 3위: 상황별 분위기 추천

> 예시: 혼밥/데이트/가족/부모님 상황별로 "분위기"가 맞는 장소 추천

#### 장점
- 메뉴보다 **"상황 연출"**이 핵심 (조용함/밝기/좌석/대화 난이도)
- 상황 버튼 UI로 진입장벽 낮음
- 1위 아이디어와 데이터 소스 동일 → **통합 개발 가능**

#### 데이터 소스
| 소스 | 수집 방법 | 데이터 |
|------|----------|--------|
| 네이버 플레이스 | 크롤링 | "조용하다/시끄럽다/테이블 간격/사진 잘 나옴" 등 분위기 표현 |
| 카카오맵 | 크롤링 | 리뷰 텍스트 |

#### MVP 범위
- 상황 버튼: 혼밥, 데이트, 가족, 회식 (4개)
- 채팅 보조로 세부 조건 파악

## 최종 추천: 통합 서비스 전략

### 컨셉
> **"내 위치 + 컨디션 + 상황"** 기반 맛집/카페 추천 서비스

### 통합 시 장점
- 1, 2, 3위 아이디어가 **동일한 데이터 소스** 사용
- 하나의 서비스로 **3가지 차별점** 제공
- 개발 리소스 효율화

### 기술 스택 제안
```
데이터 수집: Python + Selenium/Playwright
백엔드: FastAPI / Node.js
AI/NLP: OpenAI API / KoNLPy
프론트엔드: React / Next.js
DB: PostgreSQL + Elasticsearch (리뷰 검색)
```

### 데이터 파이프라인
```
1. Naver Place API → 기본 정보 수집
2. 네이버 플레이스 크롤링 → 리뷰 텍스트 수집
3. NLP 처리 → 키워드 추출 (맛/분위기/컨디션)
4. 벡터 DB 저장 → 유사도 검색
5. LLM → 추천 이유 생성
```

### 5주 MVP 로드맵

| 주차 | 작업 내용 |
|:---:|----------|
| 1주 | 데이터 크롤링 파이프라인 구축, 지역 1개 데이터 수집 |
| 2주 | NLP 키워드 추출, 태깅 시스템 구축 |
| 3주 | 추천 알고리즘 개발, API 서버 구축 |
| 4주 | 프론트엔드 UI 개발, 채팅 인터페이스 |
| 5주 | 통합 테스트, 버그 수정, 데모 준비 |
