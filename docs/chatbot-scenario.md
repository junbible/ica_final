# 냠(NYAM) 챗봇 시나리오 문서

> 프론트엔드 개발용 대화 흐름 및 멘트 정의
> 작성일: 2025.01

---

## 1. 개요

### 1.1 서비스 컨셉
- **No Text, Only Swipe**: 텍스트 입력 없이 스와이프만으로 추천
- **브랜드 보이스**: 귀엽고 명쾌한 해결사 ("고민 말고 냠!")
- **말투**: 친근한 반말 + 이모지 활용

### 1.2 화면 흐름
```
S-01 메인 → S-02 위치확정 → S-03 스와이프 질문 → S-04 처리 → S-05 결과
                                                          ↓
                                                    S-06 실패
```

## 2. 화면별 시나리오

### S-01. 메인 진입

| 구분 | 내용 |
|------|------|
| **봇 첫인사** | "오늘 뭐 먹을지 고민이야? 🤔" |
| **서브 멘트** | "컨디션에 딱 맞는 메뉴, 내가 찾아줄게!" |
| **CTA 버튼** | "추천 시작하기" |

**대안 멘트 (랜덤 or A/B 테스트용)**
- "고민 말고 냠! 🍽️"
- "배고파? 내가 정해줄게!"
- "오늘 컨디션 어때? 맛있는 거 찾아줄게!"

#### 2.2 권한 허용 시

| 구분 | 내용 |
|------|------|
| **성공 멘트** | "좋아! 현재 위치 기준으로 찾아볼게 🔍" |

#### 2.3 권한 거부 시

| 구분 | 내용 |

### S-03. 컨디션 입력 (스와이프)

#### 3.1 시작 멘트

| 구분 | 내용 |
|------|------|
| **안내 멘트** | "지금 컨디션 알려줘! 👇" |
| **사용법** | "← 아니 / 응 →" |
| **진행 표시** | "1/4" (하단 인디케이터) |

#### 3.2 질문 카드 (4개)

| 순서 | 질문 | 좌측 (No) | 우측 (Yes) |
|------|------|-----------|------------|
| **Q1** | "자극적인 게 땡겨? 🌶️" | 담백한 게 좋아 | 매콤하게! |
| **Q2** | "따뜻한 게 먹고 싶어? 🍜" | 시원한 게 좋아 | 따끈하게! |
| **Q3** | "가볍게 먹을래? 🥗" | 든든하게! | 가볍게! |
| **Q4** | "국물이 필요해? 🍲" | 없어도 돼 | 필요해! |

#### 3.3 스와이프 피드백 (선택사항)

| 선택 | 피드백 (짧은 텍스트 or 이모지) |
|------|------------------------------|
| Q1 → Yes | "매운 맛 좋지! 🔥" |
| Q1 → No | "순한 맛도 좋아 ☺️" |
| Q2 → Yes | "따끈한 거 찾아볼게! 🍜" |
| Q2 → No | "시원한 거 좋지! 🧊" |
| Q3 → Yes | "가볍게 가자! 🌿" |
| Q3 → No | "든든하게 먹어야지! 💪" |
| Q4 → Yes | "국물 최고! 🍲" |
| Q4 → No | "국물 없이 가자! 🍖" |

## 4. 마이크로카피 가이드

### 4.1 톤앤매너

| 원칙 | 예시 |
|------|------|
| **친근한 반말** | "뭐 먹을래?" (O) / "무엇을 드시겠습니까?" (X) |
| **짧고 명확하게** | "매콤하게!" (O) / "매콤한 음식을 원하시나요?" (X) |
| **이모지 활용** | 문장 끝에 1개 정도 사용 |
| **긍정적 표현** | "찾아볼게!" (O) / "검색합니다" (X) |

### 4.2 피해야 할 표현

- 존댓말 ("~하시겠습니까?")
- 딱딱한 시스템 메시지 ("오류가 발생했습니다")
- 과도한 이모지 (문장당 3개 이상)
- 길고 복잡한 설명
