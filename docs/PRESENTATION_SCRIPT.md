# 발표 대본 — nyam! 프로젝트 진행 현황

> 예상 소요 시간: 5~7분

---

## 1. 인트로 (30초)

안녕하세요, nyam! 프로젝트 진행 현황 발표하겠습니다.

nyam!은 "오늘 뭐 먹지?" 고민을 해결해주는 서비스입니다. 사용자의 컨디션 — 피곤하다, 숙취가 있다, 스트레스 받는다 — 이런 상태를 AI 챗봇에게 말하면, 맞춤 메뉴를 추천하고 근처 실제 맛집까지 찾아줍니다.

현재 Railway에 배포되어 있고, PWA로 모바일 설치도 가능한 상태입니다.

---

## 2. 주요 기능 소개 (1분 30초)

현재 v1.4까지 개발이 완료되었고, 크게 네 가지 핵심 기능이 있습니다.

**첫째, AI 챗봇입니다.** OpenAI GPT-4o-mini를 사용하고 있고요, 사용자가 "피곤해요"라고 하면 삼계탕, 국밥 같은 메뉴를 이미지 카드와 함께 추천해줍니다. 위치를 말하면 카카오 API로 실제 맛집을 검색해서 지도와 함께 보여줍니다. 이번 버전에서는 "홍대 파스타"처럼 음식과 위치를 한 번에 입력하는 직접 검색도 추가했습니다.

**둘째, 메인 페이지입니다.** 시간대별 인사말, 컨디션 바로가기, 지역별 맛집, 테마 컬렉션 등을 제공합니다. 모든 데이터가 카카오 Local API 실시간 데이터입니다.

**셋째, 맛집 상세 페이지입니다.** 카카오 플레이스에서 가져온 실제 리뷰, 영업시간, 지도를 보여주고, 전화 걸기나 카카오맵으로 바로 이동할 수 있습니다.

**넷째, 카카오 소셜 로그인과 즐겨찾기입니다.** 로그인하면 마이페이지에서 즐겨찾기한 맛집 목록을 관리할 수 있습니다.

---

## 3. 기술 스택 (30초)

프론트엔드는 React + TypeScript + Vite로 구성했고, UI는 shadcn/ui와 Tailwind CSS를 사용합니다. PWA를 지원해서 앱 설치와 오프라인 캐싱이 됩니다.

백엔드는 FastAPI이고, OpenAI API로 대화를 처리하고, 카카오 Local API로 맛집을 검색합니다. 인증은 JWT 기반이고, 데이터베이스는 Supabase PostgreSQL을 사용합니다. Railway에 Docker로 배포하고 있습니다.

---

## 4. 이번 주 작업 — v1.4.0 (1분 30초)

이번 주 가장 큰 변경은 **카카오 API 실데이터 연동**입니다.

기존에는 프론트엔드에 18개 가짜 맛집 데이터가 하드코딩되어 있었는데, 이걸 전부 카카오 Local API 실시간 검색으로 교체했습니다. 백엔드에 맛집 전용 API 모듈을 새로 만들고, 프론트엔드도 전 페이지를 API 호출 방식으로 전환했습니다.

두 번째로 **안정성을 크게 개선**했습니다. 카카오맵 SDK와 React가 같은 DOM을 건드리면서 발생하는 크래시 문제가 있었는데, 맵 컨테이너를 React 관리 영역과 완전히 분리해서 해결했습니다. 또 채팅에서 에러가 나면 전체 앱이 멈추는 문제가 있었는데, 채팅 전용 Error Boundary를 추가해서 채팅 영역만 에러 처리되도록 했습니다.

그 외에 채팅 입력 커서가 사라지는 문제, 메뉴 이미지가 깨지는 문제 등 여러 버그를 수정했습니다.

---

## 5. 기술적 챌린지 (1분)

개발하면서 겪은 기술적 어려움을 두 가지 공유드립니다.

**첫째, 카카오 API의 한계입니다.** 카카오 Local API는 맛집 이름, 주소, 좌표만 주고 메뉴, 리뷰, 영업시간, 이미지는 제공하지 않습니다. 그래서 카카오 플레이스 비공식 API를 활용해서 리뷰와 영업시간을 가져오고, og:image 태그에서 대표 이미지를 추출하는 방식으로 해결했습니다.

**둘째, 외부 SDK와 React의 DOM 충돌입니다.** 카카오맵 SDK가 지도를 렌더링할 때 컨테이너의 innerHTML을 통째로 교체하는데, 같은 곳에 React가 관리하는 로딩 텍스트가 있으면 나중에 React가 해당 노드를 제거하려 할 때 크래시가 납니다. 이걸 맵 전용 div와 React UI를 분리하고, 조건부 렌더링 대신 CSS display로 전환해서 해결했습니다.

## 6. 데모 (1분)

*(화면 공유하며)*

간단히 데모 보여드리겠습니다.

1. 메인 페이지입니다. 지역별 맛집, 컬렉션 등이 카카오 실데이터로 표시됩니다.
2. 챗봇을 열어서 "피곤해요"라고 입력하면 — 삼계탕, 비빔밥, 국밥이 이미지와 함께 추천됩니다.
3. "강남"이라고 입력하면 — 강남 근처 실제 삼계탕 맛집 5개가 지도와 함께 나옵니다.
4. 맛집을 클릭하면 상세 페이지로 이동합니다. 실제 리뷰, 영업시간, 지도가 보입니다.
5. "홍대 파스타"처럼 직접 검색도 가능합니다.

## Q&A 예상 질문

**Q: 카카오 API 키 없이도 동작하나요?**
A: 네, API 키가 없으면 자동으로 목업 데이터로 전환됩니다. 개발 환경에서도 정상 동작합니다.

**Q: 챗봇의 추천 로직은 어떻게 되나요?**
A: 사용자 메시지에서 컨디션 키워드(피곤, 숙취 등)를 감지하고, 컨디션별로 매핑된 음식 키워드로 카카오 API를 검색합니다. 대화 자체는 OpenAI가 처리합니다.

**Q: 비공식 API 사용에 문제는 없나요?**
A: 카카오 플레이스 웹페이지의 공개 데이터를 읽는 수준이라 현재까지 문제없이 동작합니다. 다만 장기적으로는 공식 API 확장이나 자체 리뷰 시스템으로 전환을 고려하고 있습니다.

**Q: PWA라서 앱스토어 배포도 가능한가요?**
A: PWA는 별도 앱스토어 등록 없이 브라우저에서 "홈 화면에 추가"로 설치할 수 있습니다. 필요하면 TWA(Trusted Web Activity)로 구글 플레이스토어 배포도 가능합니다.
